# Reports — maintainer documentation

Material intended for people maintaining or extending the reports
package, rather than for end users of `spectifyr` or the
`ReportServiceClient` library.

## What's here

- **[architecture.md](architecture.md)** — the report service
  architecture: storage, HTTP API, retention.
- **[configuration-design.md](configuration-design.md)** — design
  document for the mode-based audit trail and the orchestrator-side
  configuration that drives it.
- **[versioning-strategy.md](versioning-strategy.md)** — how the
  server and the client library are versioned in this single npm
  package.
